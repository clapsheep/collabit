### ☀️ 시작 전

#### [debounce, throttle]

- 진행중인 사이드 프로젝트에서 사용자의 편의를 위해 입력과 동시에 검색 api를 요청하는 기능을 구현, debounce 없이 구현했더니 지나치게 많은 요청과 함께 오히려 정신없는 사용자의 경험이 나타났다. 기존에는 단순히 성능 최적화를 위한 기술이라 생각했지만, 결국 그 성능 최적화가 모여서 사용자의 경험으로 반영된다는 생각을 하게 되었다.

- debounce? throttle?
  > debounce는 정해진 시간동안 발생한 연속된 이벤트를 묶어 마지막 1 번만 발생시키는 것이고, throttle은 일정 주기마다 이벤트를 발생시키는 것으로 작동방식이 약간 다르다.
- 활용 방안

  > debounce : 검색창 자동완성, 창크기에 조절하는 반응 이벤트, 저장버튼 연속클릭방지, 입력 폼의 실시간 유효성 검사

  > throttle : 무한 스크롤 구현, 실시간 게임에서의 키보드/마우스 입력, 차트나 그래프의 실시간 데이터 업데이트, 브라우저 리사이즈 이벤트

- debounce 직접구현

```ts
const debounce = <T>(callback: (arg: T) => void, duration: number) => {
  let timer: NodeJS.Timeout | undefined;
  return (arg: T) => {
    if (timer) {
      clearTimeout(timer);
    }
    timer = setTimeout(() => {
      callback(arg);
    }, duration);
  };
};

export default debounce;
```
